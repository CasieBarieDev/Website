CasieBounce




The innovative plugin that adds a whole new dimension to movement and exploration!

Designed for fun and excitement, CasieBounce allows players to experience the thrill of bouncing across terrains, leaping to incredible heights, and exploring the world like never before. Whether you’re building a parkour challenge, creating a fun mini-game, or just want to add some extra flair to your server, CasieBounce is the perfect addition.

Setup

  1. Download the plugin from Spigot .
    • Optional: Download WorldGuard .
    • Optional: Download PlaceholderAPI .
  2. Put the '.jar' file in your 'plugins' folder.
  3. Restart your server to initialize the plugin.
  4. Using WorldGuard (Optional)
    • In the config file, set 'WorldGuardFlags' to true.
    • Restart your server again to register the flags!
    • You can now use the following flags to adjust region-specific settings: 'cb-bounceforce, cb-deathmessage, cb-falldamage, cb-bouncesound, cb-bounceblocks, cb-stopwhencrouch, cb-requirepermission, cb-isblockblacklist'.
    • Any unspecified settings will default to the global config values.
  5. Adjust the settings in the configuration file to your preferences.
  6. Use /cb ReloadConfig to apply changes. Any errors will be displayed in the console.
  7. HAPPY BOUNCING!

Commands

  • /cb ReloadConfig
    • Reloads the configuration file to apply any changes.
  • /cb Info
    • Displays an informational message about the plugin.
  • /cb GetErrors
    • Checks the configuration for errors and displays them if found.
  • /cb GetRegionSettings
    • Shows the bounce settings for the region you are currently standing in.
  • /cb ResetData
    • Deletes player data or region data from the database.

Permissions

  • cb.bounce
    • Grants the player permission to bounce in all regions where 'RequirePermission' is set to true.
  • cb.bounce.‹region›
    • Grants the player permission to bounce specifically in the ‹region› where 'RequirePermission' is set to true.
  • cb.admin
    • Grants the player access to all administrative commands.

Configuartion

General Settings

  • WorldGuardFlags: Boolean
    • Toggles the use of WorldGuard flags.
    • Once enabled, a full server restart is required to register the flags in WorldGuard.

Default Settings

These settings will get used when WorldGuard is not enabled or when the setting is not specified in the region:

  • BounceForce: Double
    • Determines the amount of force applied when bouncing.
  • BounceSound: String
    • Specifies the sound to play when bouncing.
    • Use CUSTOM: at the start for custom sounds.
    • Set to NONE to disable the sound.
  • BouncePrize: String
    • Defines a prize for players each time they bounce.
    • Format: MONEY@{VALUE} or ITEM@{VALUE} or PERMISSION@{VALUE}.
    • Set to NONE to disable
  • StopWhenCrouch: Boolean
    • Specifies whether bouncing stops when crouching.
  • FallDamage: Boolean
    • Enables or disables fall damage while bouncing.
  • DeathMessage: String
    • Sets the death message displayed when a player dies due to bouncing.
    • & and #hex codes are supported)
  • RequirePermission: Boolean
    • Whether players need the permission cb.bounce to bounce.
  • BounceBlocks: List
    • A list of blocks where players can bounce.
  • IsBlockBlacklist: Boolean
    • Defines whether the BounceBlocks list acts as a blacklist.
    • When true players can bounce on every block except those in the BounceBlocks list.

Placeholders

If you have PlaceholderAPI installed, you can enhance your server experience by accessing detailed bounce statistics through placeholders.

With these placeholders, you can display bounce-related data in creative and dynamic ways. For example, you can create leaderboards to highlight top bouncers, making it simple to track and celebrate player achievements across your server.
